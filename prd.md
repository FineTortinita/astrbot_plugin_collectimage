# 产品需求文档 (PRD): astrbot_plugin_collectimage

## 1. 项目背景
开发一个名为 `astrbot_plugin_collectimage` 的插件，用于自动监听群聊中的图片消息，利用视觉大模型（VLM）进行内容分析，并实现图片的本地持久化存储及结构化数据记录。

## 2. 核心功能设计 (User Stories)

### US-001: 自动监听与群组过滤
**描述**：作为管理员，我希望插件能自动监听特定群组的图片消息，以便无感化地收集数据。
**验收标准**：
- 插件实时监听 `GroupMessageEvent`。
- 仅处理 `config.json` 中 `allowed_groups` 列表内的群组。
- 若白名单为空，默认不处理任何群组消息。

### US-002: 表情包与元数据直接过滤
**描述**：作为用户，我希望插件能自动过滤表情包，以避免数据库中充斥无意义的垃圾数据。
**验收标准**：
- 识别 `napcat` 等适配器的元数据（如 `sticker` 类型或 `sub_type: 1`）。
- 判定为表情包后立即终止流程，不下载、不分析、不存储。
- 提供视觉二次校验：若模型判定 `is_meme` 为真，则立即删除临时文件。

### US-003: 视觉大模型 (VLM) 结构化分析
**描述**：作为系统，我需要调用视觉大模型对图片进行理解，并提取结构化标签和描述。
**验收标准**：
- 优先调用 AstrBot 核心配置的默认视觉模型。
- 支持在插件配置中覆盖独立 API（OpenAI/Gemini 兼容）。
- 必须返回 JSON 格式，包含 `description`, `tags` (数组), `is_meme` (布尔值)。

### US-004: 图像持久化与 SQLite 记录
**描述**：作为系统，我需要将分析结果和原始图片持久化存储，以便后续检索。
**验收标准**：
- 图片保存至 `data/plugins/astrbot_plugin_collectimage/images/`。
- 命名格式严格遵循 `时间戳_群号_发送者ID.ext`。
- 使用 SQLite 记录元数据，`local_path` 字段必须指向本地保存的相对路径。

### US-005: 标签检索交互指令
**描述**：作为用户，我希望通过指令检索特定类型的图片，以便快速找到历史素材。
**验收标准**：
- 实现 `/search_tag <标签>` 指令。
- 插件检索数据库中包含该标签的记录。
- 返回图片的分析摘要并尝试发送对应的原始图片。

### US-006: 数据库统计面板展示
**描述**：作为管理员，我希望查看插件的运行概况，以便了解数据收集的进度。
**验收标准**：
- 实现 `/image_stats` 指令。
- 显示已收集图片总数。
- 展示热门标签 Top 5 以及最活跃的群组统计信息。

## 3. 配置规范 (`config.json`)
```json
{
  "allowed_groups": ["123456789"],
  "vision_model_endpoint": "",
  "vision_model_api_key": "",
  "enable_sticker_filter": true
}
```

## 4. 技术约束
- **开发规范**：严格遵循 AstrBot 插件开发标准，使用 `@register`, `@command` 装饰器。
- **异步设计**：网络请求与数据库操作必须异步，不得阻塞机器人主线程。
- **元数据适配**：需兼容 `napcat` 的消息段结构，精准提取图片 URL。

## 5. 验收汇总
- [ ] 插件能自动捕获白名单群组的图片消息。
- [ ] 表情包（Sticker）被正确识别并跳过。
- [ ] 图片成功保存至指定目录，且命名符合规范。
- [ ] SQLite 数据库记录包含正确的本地路径及分析结果。
- [ ] 支持通过标签搜索历史图片。
- [ ] 提供统计面板展示收集概况。
